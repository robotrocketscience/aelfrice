# Design specs

Internal design notes and per-feature specs. These are written for contributors working on a specific subsystem; they are not user docs.

Filename conventions:

- `feature-*.md` — feature specs, usually paired with a tracking issue.
- `v2_*.md` / `v3_*.md` — version-scoped design proposals (some shipped, some superseded).
- everything else — implementation notes for a specific module or behavior (e.g. `bayesian_ranking.md`, `bfs_multihop.md`).

Specs may be out of date relative to shipped code; treat the source as the truth and the spec as the historical intent.
