# Changelog — v2.x

All notable changes to the v2.x release line are documented in this file.
The full changelog index lives in [`../CHANGELOG.md`](../CHANGELOG.md).

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [2.1.0] - 2026-05-09

Minor release. Two retrieval-lane flag flips clear their reproducibility gates and ship default-on (#154); the auto-capture pipeline that the README v1.2.0 roadmap marketed as shipped now actually wires by default on bare `aelf setup` (#529); the `/aelf:upgrade` flow gains end-to-end robustness against `aelf upgrade-cmd --check` regressions (#530 + #522); and the HRR structural-query lane drops to a 4×-cheaper default dimensionality (#538).

### Performance

- **`hrr.DEFAULT_DIM` flips 2048 → 512** ([#538](https://github.com/robotrocketscience/aelfrice/issues/538)). Aligns the HRR default dimensionality with the lab campaign R4 verdict ("adopt dim=512 default; dim=2048 escape hatch") that produced the spec at `docs/hrr_structural_query_lane.md`. ~4× memory footprint reduction for the structural-query lane: ~200 MB at N=50k vs the prior ~800 MB. Capacity per Plate's bound stays well above realistic per-belief role multiplicity (~57 retrievable bound pairs at dim=512 vs ~5 typical outgoing edges per belief). Set `dim=2048` explicitly via the `HRRStructIndex(dim=...)` kwarg for high-multiplicity corpora.

### Changed

- **`use_heat_kernel` and `use_hrr_structural` flip to default-on** ([#154](https://github.com/robotrocketscience/aelfrice/issues/154)). Both retrieval-lane flags shipped opt-in earlier (heat-kernel #150 in v1.6, HRR structural-query lane #152 in v1.7) pending the calibrated reproducibility-harness gate. That gate cleared at 11/11 via [PR #489](https://github.com/robotrocketscience/aelfrice/pull/489) closing #437, so the #154 composition tracker flips the defaults per its status banner. Precedence (env > kwarg > TOML > default) is unchanged; only the default values flip `False` → `True`. Reversible via `[retrieval] use_heat_kernel = false` / `use_hrr_structural = false` in `.aelfrice.toml` (or `AELFRICE_HEAT_KERNEL=0` / `AELFRICE_HRR_STRUCTURAL=0`) for parity with the v2.0.x ranking.
- **`aelf setup` wires the auto-capture pipeline by default** ([#529](https://github.com/robotrocketscience/aelfrice/issues/529)). Bare `aelf setup` now installs `--transcript-ingest` (UserPromptSubmit + Stop + PreCompact + PostCompact transcript-logger), `--commit-ingest` (PostToolUse:Bash commit-message ingest), and `--session-start` (SessionStart locked-belief injection) alongside the read-side `UserPromptSubmit` retrieval hook — closing the gap between what the README v1.2.0 roadmap marketed as shipped and what fresh installs actually wired. Three flags flip from `store_true` to `argparse.BooleanOptionalAction(default=True)`; opt-out via `--no-transcript-ingest`, `--no-commit-ingest`, `--no-session-start`. `aelf unsetup` mirrors symmetrically. `--rebuilder` and `--search-tool` remain opt-in (separate scope). `docs/INSTALL.md` § "Hooks installed by `aelf setup`" rewritten as a six-row table with an explicit privacy note for default-on transcript-ingest (no PII scrubber on JSONL ingest; opt out if you paste secrets in chat).

### Added

- **#291 query-strategy uplift bench gate scaffold + driver** ([#291](https://github.com/robotrocketscience/aelfrice/issues/291), #527). New `run_query_strategy_uplift` driver + bench-gate harness for measuring NDCG@k uplift from query-strategy variants on the labelled corpus. Skip-on-no-corpus pattern; no operator action required absent `AELFRICE_CORPUS_ROOT`.
- **#435 doc-linker uplift bench gate** ([#435](https://github.com/robotrocketscience/aelfrice/issues/435)). New `run_doc_linker_uplift` driver for the doc-linker bench gate, same skip-on-no-corpus pattern.
- **#433 vocab_bridge uplift bench gate scaffold + driver** ([#433](https://github.com/robotrocketscience/aelfrice/issues/433)). New `run_vocab_bridge_uplift` driver in `tests/retrieve_uplift_runner.py` mirroring `run_doc_linker_uplift` / `run_query_strategy_uplift`, plus the strict-NDCG ship-gate scaffold in `tests/bench_gate/test_vocab_bridge_uplift.py` (skip-if-runner-absent + skip-if-no-row-has-`expected_top_k`). Measures NDCG@k uplift from `use_vocab_bridge=True` on the labelled corpus. Skip-on-no-corpus pattern; no operator action required absent `AELFRICE_CORPUS_ROOT`.

### Fixed

- **`/aelf:upgrade` slash uses the no-flag `aelf upgrade-cmd` form for cross-CLI compatibility** ([#530](https://github.com/robotrocketscience/aelfrice/issues/530)). The v2.0.1 slash file invokes `aelf upgrade-cmd --check` and parses `run:` out of the output, but v2.0.1's CLI suppresses `run:` under `--check` (the #522 fix landed in this same release). On v2.0.1 installs the slash silently does nothing — the user reads "(installed: <old>)" out of the partial output and concludes the upgrade did not happen. Step 1 now invokes the no-flag form, which has emitted `run:` on every released CLI back to v1.0.1, making the slash robust against the same flag-gated regression class. Companion to #522 (CLI symmetry); #522 prevents recurrence post-v2.1.0, this prevents the parse-step blackout for any pre-#522 user who lands the new slash via `aelf setup`. v2.0.1 users currently stranded should run their install-method's upgrade command directly (`pipx upgrade aelfrice`, `uv tool upgrade aelfrice`, etc.) and then `aelf setup` to redeploy the fixed slash bundle.
- **`aelf upgrade-cmd --check` emits the `run:` line on update-available** ([#522](https://github.com/robotrocketscience/aelfrice/issues/522), [PR #525](https://github.com/robotrocketscience/aelfrice/pull/525)). The `/aelf:upgrade` slash file shipped in v2.0.1 (#513) parses a `run: <command>` line out of `aelf upgrade-cmd --check` to know which install-method-aware command to execute. Pre-fix, the `--check` form short-circuited that line and the install-context note, so the slash file's parse step silently produced nothing on installs that had received the v2.0.1 slash bundle out-of-band against an older CLI. The `if not args.check` guard is removed; both `--check` and no-flags forms now emit identical output on update-available, and the "up to date" path is unchanged (no `run:` line, no asymmetry). Three regression tests cover the byte-for-byte agreement.
- **Bug-report issue template accepts inline screenshots** ([#531](https://github.com/robotrocketscience/aelfrice/issues/531)). The `what-happened` field had `render: shell`, which wraps content in a code block and prevents inline image rendering. Drop `render: shell` so dragged/pasted screenshots render as images; "Minimal reproduction" keeps `render: shell` for exact CLI commands.
- **`aelf core` no longer crashes on α+β==0 beliefs** ([#439](https://github.com/robotrocketscience/aelfrice/issues/439)). Two divide-by-zero paths in the `aelf core` CLI surface: (1) the sort path in `_emit_core` dereferenced `alpha / (alpha + beta)` unconditionally, so a malformed belief with `alpha = beta = 0` raised `ZeroDivisionError` and crashed the command; (2) three sites in `_qualifies_core` / `_emit_core` (JSON + text paths) divided `alpha/ab` guarded only by `ab >= args.min_alpha_beta`, so `--min-alpha-beta 0` (a documented valid value) admitted α+β==0 beliefs that then crashed the same dereference. Both paths now guard with `alpha / ab if ab else 0.0`, matching the pre-existing JSON-path guard.
- **MCP `tool_lock` returns a structured error on empty derivation** ([PR #512](https://github.com/robotrocketscience/aelfrice/pull/512)). `tool_lock` had `assert derived.belief is not None` directly on the hot path. When the derivation worker rejected the input (empty post-strip, question-shaped, classifier sets `persist=False`), the assert fired and the MCP tool surface crashed with an unhandled `AssertionError` rather than returning a result the caller could parse. Replaced with a structured error response.
- **`vocab_bridge.py` storage-cost docstring cites `dim=512` default** ([#541](https://github.com/robotrocketscience/aelfrice/issues/541), [PR #543](https://github.com/robotrocketscience/aelfrice/pull/543)). After #538 flipped `DEFAULT_DIM` 2048→512, the `VocabBridge` storage-cost docstring at `src/aelfrice/vocab_bridge.py:113` still cited `dim=2048` as the implied default for the `(N_surfaces, dim)` storage estimate. Updated to cite `dim=512` (~40 MB at N=10k) as the primary number with `dim=2048` (~160 MB) shown as the escape-hatch path, mirroring the format used in `hrr_index.py:107-110`.

## [2.0.1] - 2026-05-09

Patch release. Ships the `/aelf:upgrade` banner-persistence fix from [#513](https://github.com/robotrocketscience/aelfrice/pull/513) so users on v2.0.0 stop seeing the upgrade nag after upgrading.

### Fixed

- **`/aelf:upgrade` banner persistence after upgrade** ([#513](https://github.com/robotrocketscience/aelfrice/pull/513)). Three changes make the upgrade UX imperative and clear the post-upgrade banner correctly: (1) `/aelf:upgrade-cmd` is renamed to `/aelf:upgrade` and rewritten as an imperative orchestrator that calls `aelf upgrade-cmd --check`, runs the install-aware command in Bash, then `aelf setup` (refresh slash bundle), then `aelf upgrade-cmd` (clear stale banner cache); (2) `detect_reachable_installs()` no longer false-positives the running interpreter's venv as a separate install under `uv run`, removing the spurious "multiple installs detected" warning; (3) the default update banner (`format_update_banner(latest)` with no `command` kwarg) now points at `/aelf:upgrade`. CLI surface unchanged: `aelf upgrade-cmd` (canonical advisory print) and `aelf upgrade` (deprecated alias) both still work.

## [2.0.0] - 2026-05-07

Reproducibility cut. v2.0 closes the v1.5–v1.7 feature-parity wave with a single tagged release. No additional product surface beyond v1.7.0 — same wheel content, same retrieval defaults, same CLI surface. v2.0.0 is the canonical reproducibility tag for the v2.0 corpus + bench evidence cut: operators who want a stable pin for the BM25F default-on flip (#154, +0.6650 NDCG@k on the v0.1 retrieve_uplift fixture) plus the v1.7.0 detector wave (#264 worker refactor, #421/#387 edge-type rerank + stale writer, #422 value-comparison, #441 confirm, #197/#201 dedup + contradiction, #228 wonder-consolidation, #383-388 edge-type ship gates) should pin `aelfrice==2.0.0` rather than `aelfrice==1.7.0`. README roadmap rows for v1.7 and v2.0 both flip to `shipped` with this tag.
