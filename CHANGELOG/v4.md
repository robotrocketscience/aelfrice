## [Unreleased]

### Added

- **Phantom promotion-opportunity detector ([#1132](https://github.com/robotrocketscience/aelfrice/issues/1132)).** The promotion-side mirror of the #980 phantom-*generation* detector. On each `UserPromptSubmit` turn, when the opt-in `[phantom_promotion]` lane is enabled, aelfrice deterministically checks whether any phantom (`origin='speculative'`) has crossed a cross-session corroboration threshold (default ≥3 corroborations across ≥2 distinct sessions, no inbound CONTRADICTS) and, if so, appends a small `<aelfrice-phantom-promotion-opportunity>` note naming the candidate(s) and their `aelf validate <id>` / `aelf lock` surface. This closes the #1125-census gap — phantoms were essentially never promoted (0 across seven real stores) not because promotion is broken but because nothing surfaced a corroborated phantom for the explicit act the ratified #229 rule requires. **Note-not-write:** origin promotion stays exactly where #229 put it (an explicit user act; a corroboration count is a *non-trigger* for the write); this lane only decides *when to prompt*, never promotes autonomously. New pure `aelfrice.phantom_promotion_opportunity` module (config resolution mirroring `[phantom_generation]`, deterministic `store.find_promotable_phantoms` detector, per-session budget + belief-id dedup in `session_ring` state, XML-escaped note) and a fail-soft `UserPromptSubmit` lane. Default-off; inert until enabled. No clock, no network, no LLM (#605-clean).
- **Keyword-triggered belief categories ([#1126](https://github.com/robotrocketscience/aelfrice/issues/1126)).** Group beliefs into named categories (repo-rules, git-workflow, prose-and-docs, …) and bind each to an activation trigger; when a category fires — it is always-on, or one of its keyword phrases appears in the prompt — the `UserPromptSubmit` hook **reranks the retrieval output** so that category's rules lead the `<aelfrice-memory>` block, prepends a one-line `<category-focus>` label, and surfaces a bounded set of members retrieval missed. The conditional, right-rule-at-the-right-moment complement to a static `CLAUDE.md`/`AGENTS.md`. It **reranks rather than injecting a second block**: R&D on the feature showed a separate block double-injects what retrieval (L0 + BM25) already returns and that category members are almost always already in the retrieval tail, so the value is prioritising + labelling the one block, not adding content. **Advisory, not enforcement** — it never blocks a tool call (per the #199 enforcement history and the fail-open hook contract); a hard-block lane is an explicit non-goal. Two additive tables (`categories` with a `trigger_json` config blob + a `belief_categories` M2M join modeled on `belief_documents`) — no `Belief` column, and additive DDL that passes the destructive-only migration-policy gate. New pure `aelfrice.category` module: deterministic keyword (case-insensitive, word-boundary, whitespace-tolerant) and cross-platform `fnmatch` glob matchers, stable trigger (de)serialization, and the default-off `is_enabled` gate (env `AELFRICE_BELIEF_CATEGORIES` > `[belief_categories] enabled` > `false`) — no embeddings, no model call, no clock (#605-clean). New `aelf category` verb (`init`/`add`/`list`/`show`/`set-trigger`/`assign`/`unassign`/`delete`) and `/aelf:category` slash command; assign at lock time with `aelf lock "<rule>" --category <name>`. `aelf category init` seeds a 5-category starter set. Membership is user-driven — no auto-classification. The tool/file trigger lanes are stored + matched but their `PreToolUse` wiring is a follow-up. Fail-soft throughout: any error leaves the retrieval hits unchanged and the hook returns exit 0.

### Changed

- **Hot-path performance overhaul ([#1135](https://github.com/robotrocketscience/aelfrice/issues/1135)).** Measured audit of the retrieval + ingest hot paths, fixed structurally (no ranking change — retrieval outputs are byte-identical on a fixed store):
  - *BM25F index persists across processes.* The default-ON BM25F lane rebuilt its index from scratch on every `retrieve()` — the `UserPromptSubmit` hook is a fresh process per prompt, so every prompt re-tokenized and Porter-stemmed the whole corpus (~584 ms at 5k beliefs, linear in corpus size). The built index is now written to a `<db-path>.bm25f` sidecar stamped with a new durable `store_generation` counter (bumped in the same transaction as every belief/edge content mutation) and reloaded in low milliseconds while the stamp matches; any content change invalidates it. Long-running processes (MCP server) additionally reuse one in-memory cache per store instead of rebuilding — and no longer leak an invalidation callback per query.
  - *Bulk ingest is no longer O(n²).* Each ingested turn snapshotted the full belief-id set and re-read its log rows to tell inserts from corroborations; the derivation worker now reports per-row `(belief_id, was_inserted)` outcomes, and the worker's unstamped-log scan got a partial index (the log grows monotonically; the unstamped set stays tiny). Per-turn cost is now flat with respect to corpus size.
  - *Write groups batch into single transactions.* A new `MemoryStore.transaction()` context manager suppresses per-call commits (measured 33× cheaper than commit-per-row for a 200-insert group); wired into the ingest turn (~8 commits → 1, and a turn is now crash-atomic), the hook's retrieval-audit / injection-event / touch loops (~45–60 commits per prompt with hits → a handful), and the deferred-feedback enqueue. `PRAGMA synchronous=NORMAL` now pairs with WAL as documented.
  - *Smaller structural fixes.* Partial index for the locked-belief tier (ran up to 3× per retrieve as a full-table scan) and an index for the `edges.type` existence probe; the `UserPromptSubmit` hook opens the store once per prompt instead of 4–6 times; `.aelfrice.toml` is parsed once per file version instead of ~24× per retrieve (the directory walk still runs, so config file creation/deletion is honoured); the v1.2 origin backfill (two full-table UPDATE statements) is now a marker-gated one-shot instead of running on every store open. The BM25 serialize format is v2 (float64 scalars) so a loaded index scores byte-identically to a built one; no v1 blobs existed.

### Fixed

- **Tiered retrieval honours the `now_ts` pin end-to-end ([#1143](https://github.com/robotrocketscience/aelfrice/issues/1143)).** Four sites in the tiered path (the γ posterior-temperature resolver and expansion gate in `retrieve_with_tiers`, and the BM25F anchor-weight resolver and `bm25_l0_ratio` signal write in `_l1_hits`) read the wall clock directly, ignoring the `now_ts` pin `retrieve_v2` accepts for deterministic replay — the same defect class as the #982/#983 temporal-decay time-bomb. Both functions now take `now_ts`, read the clock at most once when unpinned, and `retrieve_v2` threads its pin down, so a pinned call performs no wall-clock read anywhere in the tiered path. Pinned by a raise-on-read seam test plus a source-level assertion covering the flag-gated γ arm.
- **`RetrievalCache` cache-hit exposure exemption specified and pinned ([#1144](https://github.com/robotrocketscience/aelfrice/issues/1144)).** `RetrievalCache.retrieve` documented an "identical contract to the free function", but a cache hit returns the memoized beliefs without repeating `retrieve()`'s retrieval-exposure enqueue (#191) — exposure is recorded once on the populating miss, not per hit. Rather than write on every hit (a `deferred_feedback_queue` insert would break the AC2 ≤ 50 µs cache-hit budget), the exemption is now documented with its rationale and pinned by tests: a hit does not re-enqueue, the populating miss enqueues once, the free function enqueues per call. Exposure is audit-only recurrence data (#1086) and `RetrievalCache` has no production call site (the hook path calls the free `retrieve()`), so the recurrence stream the #1086 consumer reads is unaffected. Documentation + test only; no runtime behavior change.
- **Reader-facing docs: prose de-formularization + residual default-flip drift ([#1141](https://github.com/robotrocketscience/aelfrice/issues/1141)).** Style pass over README and the concepts docs (drop repeated bold-led bullet formulas, unbold sentence-lead paragraphs, fix a "two recovery angles" lead on a three-item list). Content fixes found on the same sweep, continuing #1137: `ARCHITECTURE.md` still showed the temporal-spine lane default-OFF (default-ON since the #1107 Phase-2 cutover), the mirror hook as env-inert (consent-gated since v4.0, #1089), and agent-context as unreleased (shipped v4.0.0); `HARNESS_INTEGRATION.md` predated the claude-memory mirror and claimed the two stores never merge (one-way consent-gated mirror since v3.7/#985 + v4.0/#1089); `RELEASING.md` said the current line is v3.x; `MCP.md`'s "CLI-only" phrasing aligned with the corrected COMMANDS.md claim.
- **Codex host-management skills now steer to `--host codex` ([#1136](https://github.com/robotrocketscience/aelfrice/issues/1136)).** The generated `$aelf-setup` / `$aelf-doctor` / `$aelf-uninstall` / `$aelf-upgrade` skills carried the bundle's bare `aelf setup` / `aelf doctor` / `aelf uninstall` run lines verbatim, which on a Codex host install, scan, or tear down another host's configuration (settings-file hooks + statusline + slash bundle) instead of `~/.codex/hooks.json` and the `$aelf-*` skills. Each of the four now carries a `<host-adapter>` note requiring the `--host codex` form, `$aelf-setup`'s description frontmatter names the codex-host artifacts, and `aelf uninstall` accepts `--host codex` so its unsetup half routes to the codex host. Same audit, same fix wave: the skill **replace** path is now marker-gated like prune/remove (a hand-authored skill colliding with a bundled name is skipped and reported, never overwritten); prune/remove half-failures (e.g. a stray file keeping a skill directory non-empty after its `SKILL.md` is unlinked) surface as `[warn]` lines and a `failed` field on `CodexSkillsResult` instead of vanishing; `aelf setup|unsetup --host codex` now exit 1 with a clear message when the skills write fails after hooks.json was written (no raw traceback); the codex CLI paths take injectable dests and gained CLI-layer tests; and the docs' tool-mapping claim is corrected (the adapter maps the `Task` fan-out tool only).
- **Corroboration-driven retention promotion no longer reaches phantoms ([#1132](https://github.com/robotrocketscience/aelfrice/issues/1132)).** `store.find_promotable_snapshots` (the `aelf doctor --promote-retention` path that flips `retention_class` snapshot→fact once a belief is corroborated ≥3× across ≥2 sessions) filtered on `retention_class` but not origin. Phantoms ingest with `retention_class='snapshot'`, so a corroborated phantom was eligible for a count-driven flip to durable `fact` retention — the exact trigger shape the ratified #229 rule rejects on the origin axis. Added `AND origin != 'speculative'` so phantom durability, like phantom trust promotion, advances only through explicit acknowledgment. Retention and origin remain orthogonal axes; neither promotes a speculative belief on a corroboration count alone.

## [4.1.0] - 2026-07-09

### Added

- **Codex host gets the `$aelf-*` skills — the port of the `/aelf:*` slash commands.** `aelf setup --host codex` now installs, alongside the hook set (#1052), the full slash-command surface as Codex **agent skills** under `~/.agents/skills/` (invoke as `$aelf-search`, `$aelf-doctor`, …, or let Codex trigger them implicitly by description). ## [4.0.0] - 2026-07-07

The production-retrieval convergence release. Several staged retrieval lanes had
been flipped default-ON in `retrieve_v2` but were inert on the live hook path —
`hook_search.retrieve()`, `context_rebuilder`, and `mcp_server` all called the
legacy `retrieve()` — so features framed as user-facing never actually reached a
running host. This release closes that gap: `retrieve()` becomes a thin adapter
over `retrieve_v2` ([#1107](https://github.com/robotrocketscience/aelfrice/issues/1107))
and four lanes graduate onto production one at a time behind their gates —
temporal-spine ([#1064](https://github.com/robotrocketscience/aelfrice/issues/1064)),
entity-persist demotion ([#1096](https://github.com/robotrocketscience/aelfrice/issues/1096)),
intentional clustering ([#436](https://github.com/robotrocketscience/aelfrice/issues/436)),
and HRR structural-query ([#152](https://github.com/robotrocketscience/aelfrice/issues/152)) —
while two staged lanes stay intentionally off (origin-tiebreak, refuted #1013;
HRR-expand, recall-neutral #1001). Alongside the cutover: a belief-curation and
junk-percolation wave (`aelf introspect` / `retire` / `restore`, exposure-is-not-
endorsement, the entity-persistence sink; #1081 / #1086 / #1096), provenance-aware
capture (claude-memory reconcile default-on, a first-class transcript source;
#1089), dispatched-subagent memory inheritance (#1068), and first-class Codex host
support (#1052–#1055).

### Added

- **HRR structural-query lane goes live on the production `retrieve()` path ([#1107](https://github.com/robotrocketscience/aelfrice/issues/1107), Phase 5; [#152](https://github.com/robotrocketscience/aelfrice/issues/152)).** The marker-routed structural lane — which detects a `<KIND>:<target_id>` query (an uppercase edge type followed by a target belief id), bypasses the BM25F + heat-kernel textual stack, and returns the graph-connected beliefs an HRR probe ranks for that edge — was resolver-default-ON in `retrieve_v2` since v2.1 (after the #154 composition tracker cleared the #437 reproducibility harness at 11/11) but inert on the live hook path, which called the legacy `retrieve()`. The #1107 shim now passes `use_hrr_structural` resolver-driven (`None` → env→TOML→default-ON) instead of hard-off, so tooling callers that issue marker queries through the production path get structural routing. The lane is **marker-routed**: on any non-marker query (the shape of a free-text hook prompt) `parse_structural_marker` returns `None` and the call falls through byte-identically, so the graduation adds only one fast-failing regex to the non-marker hot path and its blast radius is confined to `<KIND>:<target_id>` callers. This adds a public on-HEAD signal — `benchmarks/hrr_structural_ablation.py`, a deterministic synthetic corpus of marker queries whose answers share no vocabulary with the marker string: recall@k **0.000 → 1.000** (textual BM25 cannot reach a structural answer; the lane routes it exactly), complementing the existing `tests/test_retrieve_v2_hrr_structural.py` IT1–IT6 wiring tests. Opt out with `AELFRICE_HRR_STRUCTURAL=0` or `[retrieval] use_hrr_structural = false`. The equivalence guard now pins four graduated lanes (spine + entity-persist + clustering + HRR-structural) live and a held lane (origin tie-break) off. Deterministic per #605.
- **Intentional-clustering lane goes live on the production `retrieve()` path ([#1107](https://github.com/robotrocketscience/aelfrice/issues/1107), Phase 4; [#436](https://github.com/robotrocketscience/aelfrice/issues/436)).** The multi-fact clustering pass — which biases the packed top-K toward distinct graph-connected clusters so a query needing two facts surfaces both, rather than K beliefs from one neighbourhood — was default-ON in `retrieve_v2` since v3.0 (after its A2 coverage / A3 non-regression / A4 latency gates) but inert on the live hook path. The #1107 shim now passes `use_intentional_clustering` resolver-driven (`None` → default-ON) instead of hard-off, so live hosts get the multi-fact coverage. Because the full A2/A3 gate corpus is lab-side and not shipped, this adds a public on-HEAD analogue — `benchmarks/intentional_clustering_ablation.py`, a deterministic synthetic multi-cluster corpus run through the same `run_clustering_uplift` driver the gate uses: cluster_coverage@k **0.500 → 1.000** with recall@k non-degrading (A2 ship condition PASS). A4 latency was cleared at v3.0 (60/60 PASS, p99 0.328 ms, ~15-30× under the 5 ms budget). Opt out with `AELFRICE_INTENTIONAL_CLUSTERING=0` or `[retrieval] use_intentional_clustering = false`. The equivalence guard now pins three graduated lanes (spine + entity-persist + clustering) live and a held lane (origin tie-break) off. Deterministic per #605.

- **Entity-persist demotion lane goes live on the production `retrieve()` path ([#1107](https://github.com/robotrocketscience/aelfrice/issues/1107), Phase 3; [#1096](https://github.com/robotrocketscience/aelfrice/issues/1096)).** The #1086 junk-percolation sink — the log-additive demotion that pushes beliefs grounded only in transient coordination tokens (bare PR/issue numbers, version tags) below those grounded in durable entities (file paths, symbols) — was default-ON in `retrieve_v2` since v4.0 but inert on the live hook path, which called the legacy `retrieve()`. The #1107 shim now passes `use_entity_persist_demote` resolver-driven (`None` → env→TOML→default-ON, the G2-cleared #1103 default) instead of hard-off, so the demotion reaches every host's `retrieve()`, `context_rebuilder`, and `mcp_server` — this is what makes the #1086 fix real for users rather than only bench/eval consumers. No-op on entity-free or well-grounded candidates; one batched `belief_entities` query, no candidate expansion. Opt out with `AELFRICE_ENTITY_PERSIST_DEMOTE=0` or `[retrieval] use_entity_persist_demote = false`. The `retrieve()`↔`retrieve_v2` equivalence guard now pins the graduated lanes resolver-driven with the remaining held lanes forced off. Deterministic per #605.

- **Production retrieval converges on `retrieve_v2`; temporal-spine lane goes live end-to-end ([#1107](https://github.com/robotrocketscience/aelfrice/issues/1107), Phase 2; [#1064](https://github.com/robotrocketscience/aelfrice/issues/1064)).** Several staged retrieval lanes were wired into `retrieve_v2()` only, so their default-ON flips changed benchmark/eval consumers but never what a live host retrieved — the production hook path (`hook_search.retrieve()`, `context_rebuilder`, `mcp_server`) still called the legacy `retrieve()`. The #1107 cutover makes `retrieve()` a thin adapter over `retrieve_v2` (Phase 1, behaviour-preserving: all six staged lanes forced off, pinned byte-identical by `tests/test_retrieve_v2_equivalence.py`), then graduates lanes onto the live path one at a time behind a hot-path latency gate. The **temporal-spine lane** ([#1064](https://github.com/robotrocketscience/aelfrice/issues/1064)) is the first to graduate: the shim now passes `use_temporal_spine` through resolver-driven (env → TOML → default) and its default is flipped **ON**, so — together with the writer flip ([#1111](https://github.com/robotrocketscience/aelfrice/issues/1111)) — the +14.6pp-coverage spine is live on every host's `retrieve()`, not just the eval surface. G3 latency stays in-band (`benchmarks/temporal_spine_latency.py`, Δp95 ≤ 50 ms); the lane is a no-op on spineless stores. Opt out with `AELFRICE_TEMPORAL_SPINE=0` or `[retrieval] use_temporal_spine = false`. The other staged lanes graduate one at a time behind their own gates (see the entity-persist entry above). Deterministic per #605.

- **Origin-priority retrieval tie-break, default-off ([#1089](https://github.com/robotrocketscience/aelfrice/issues/1089), axis 2).** When two retrieval candidates tie on relevance, the higher-trust *origin* now wins — so a belief curated from a claude-memory `user`/`feedback` fact file (`origin=user_validated`) outranks one auto-captured from a chat transcript (`origin=user_transcript`). It is a pure within-tier **tie-break**, never a primary rerank term: the origin key sits *between* the relevance score and the id tie-break, so relevance always dominates and byte-identical behaviour is preserved when the flag is off. This is deliberately not a resurrected origin *rerank lane* — that was refuted on LoCoMo in #1013, where the failure was a BM25 *recall* limit (the trusted fact never became a candidate), which reranking cannot fix. The priority ladder mirrors `contradiction.precedence_class` (a sync-guard test keeps the two copies aligned) and is applied in both ranked candidate tiers: the L1 FTS rerank (`_l1_hits`) and the L2.5 entity-index overlap (`MemoryStore.lookup_entities`), so it covers both keyword-rerank and exact-entity-match hits. Flag-gated (`AELFRICE_ORIGIN_TIEBREAK` / `use_origin_tiebreak`), deterministic per #605; `benchmarks/origin_tiebreak_ablation.py` is the offline ablation. The mechanism lands default-off; the default-ON flip is gated on a LoCoMo retrieval no-regression run (note that a single-provenance corpus like LoCoMo shares one origin tier, so the tie-break is inert there and cannot regress recall) and is a separate operator call.

- **`aelf introspect`: a native honest-signal view over stored beliefs ([#1081](https://github.com/robotrocketscience/aelfrice/issues/1081)).** Answers "look at my conversations and analyse the beliefs it extracted" without hand-rolled SQL. The command groups the active beliefs by ingest session (or `--by project`) and, for each, surfaces the signals that already live in the store but were never shown together: posterior mean μ and its evidence weight (α+β), **recurrence** (corroboration count, labelled explicitly as recurrence — *not* truth, since a junk line re-captured every session scores high on it alone; #1086), **grounding** (the #1096 entity-persistence axis — `durable` / `ephemeral` / `neutral` — i.e. standalone-meaningful vs context-bound), floated-vs-decided **status** read off `RESOLVES` / `POTENTIALLY_STALE` edges, and a **noise** flag (the #1081 `is_stranded_capture_noise` predicate — orphan headers and shell echoes). Stranded-capture rows float to the top of each group as the prime retire candidates. Read-only and deterministic per #605 (pure counts and edge lookups, no model, no clock); curation stays in the verbs the footer points at. Flags: `--session` / `--project` to scope, `--only-noise` for the retire shortlist, `--limit N` (0 = no cap), `--json`.

- **`aelf retire` / `aelf restore`: reversible soft-delete curation ([#1081](https://github.com/robotrocketscience/aelfrice/issues/1081)).** The gentle sibling of `aelf delete`. `retire` sets `valid_to` so a belief drops out of retrieval and FTS search while its evidence trail — edges, entity index, corroborations — is preserved, and `restore` (the previously-missing inverse of `soft_delete_belief`) clears `valid_to` and re-inserts the pruned FTS row. This makes the store's soft-delete reversible end-to-end, so curation off the `introspect` view is a low-friction, undoable pass rather than a permanent hard-delete. `retire` writes a `user_retired` / `user_retired_force` audit row (valence −1) and honours the lock guard (`--force` to retire a user-locked belief); `restore` writes `user_restored` (valence +1). Both honour the #661 read-only-federation contract. Already-retired retire and already-active restore are no-ops.

- **Entity-persistence demotion lane, default-off ([#1096](https://github.com/robotrocketscience/aelfrice/issues/1096)).** A deterministic organic sink for the belief store's #1086 percolation problem (junk ranks up, not down): a mild log-additive term in the L1 reranker that demotes beliefs with low *referential grounding* — content that grounds only to transient coordination tokens (bare PR/issue numbers, version/branch tags) rather than durable entities (file paths, error codes, symbol identifiers). The score is `S1 = durable / (durable + transient + 1)` read from the existing `belief_entities` index (`MemoryStore.entity_persistence_scores`, one batched query over the candidate set); the penalty `min(0, log(S1 + ε))` is a pure demotion (well-grounded beliefs are neutral, never boosted) and is applied only to entity-bearing candidates, so entity-free durable content (docstrings, formulae) is never touched. Motivated by a `/aelf:wonder` pass that ruled out HRR (a container algebra whose only route to semantics is a corpus-derived space — an embedding, out of #605) and a measured experiment on 118 hand-labeled coordination beliefs: entity-persistence separates durable content from ephemeral status at S1 mean 0.56 vs 0.06, lifting the durable-above-ephemeral ranking AUC from 0.48 (posterior-only) to 0.87. It is too lossy to gate deletion (so the #1081 cleanup left this class in place) but strong enough to rank on. Flag-gated (`AELFRICE_ENTITY_PERSIST_DEMOTE` / `use_entity_persist_demote`), byte-identical when unset, deterministic per #605; `benchmarks/entity_persist_ablation.py` is the offline ranking ablation. The default-ON flip is gated on a retrieval-bench no-regression check (G2) and is a separate operator call.

- **`aelf context <belief_id>`: best-effort source-turn recovery for context-loss ([#1081](https://github.com/robotrocketscience/aelfrice/issues/1081)).** Verbatim sentence-splitting at ingest can strand a bald claim away from the clause that scoped it (a statement meant as "*only time* averages, *in the coach API*" persisting as "All averages are not useful."), and injection shows only the bald sentence. This read-only diagnostic recovers the original context best-effort by unioning two substrates already in the store: a `turns.jsonl` join (the belief's `session_id` + a content-substring match recovers the full source turn, whose other sentences carry the scope) and outbound `DERIVED_FROM` `anchor_text` (adjacent-turn context aelfrice already wires). It reports honestly when neither matched — measured coverage on a real 26k-belief store is ~36% (join) / ~28% (anchor) of transcript beliefs, and onboard/scanner beliefs have no originating turn at all, so this is a recovery aid, not a guarantee. No schema change, no ingest/derive change (so no replay-soak exposure), deterministic per #605; the semantic judgment of whether the recovered scope changes how to act on the belief stays with the consuming agent. Hidden subcommand (a diagnostic lens, not a workflow verb).

- **claude-memory ingested automatically as a default-on source ([#1089](https://github.com/robotrocketscience/aelfrice/issues/1089), axis 1).** The #985 write-through mirror only ingested memory fact files the agent wrote *in-session* — pre-existing files, and any write the hook didn't observe, never reached the belief graph, and the mirror was opt-in besides. A new full-set reconcile sweep (`claude_memory_reconcile.reconcile_claude_memory`) ingests the project's entire `.../memory/<name>.md` set through the same extracted `ingest_memory_text` the hook now uses (so the two can't drift from the #985 frontmatter → origin/prior mapping). Following the G4 temporal-spine pattern ([#1064](https://github.com/robotrocketscience/aelfrice/issues/1064)), `aelf setup` runs it once per project — the **consent event** — but only once the project has a claude-memory dir, so a fresh host that hasn't adopted the auto-memory tool is untouched. The consent event is announced on stderr and flips the mirror on for that project (`is_mirror_enabled` reads the per-project sentinel kept beside the store); an explicit `AELFRICE_MIRROR_CLAUDE_MEMORY=0` / `[memory] mirror_claude_memory=false` opt-out always wins. `aelf doctor` gains a `claude-memory mirror:` row and `aelf reconcile-claude-memory [--project] [--force]` re-runs the sweep manually (idempotent — a re-run corroborates, never duplicates). One-way and non-authoritative per #985; aelfrice never writes back to the memory files. Ranking curated memory above conversational capture (axis 2) is a separate follow-up; this does not close #1089.

- **Dispatched subagents now inherit memory context ([#1068](https://github.com/robotrocketscience/aelfrice/issues/1068)).** Workers spawned via agent dispatch previously ran blind to the belief store — no SessionStart baseline, no per-prompt retrieval, and no locked constraints crossed the dispatch boundary. A new `PreToolUse:^(Agent|Task)$` hook (`aelf-agent-context-hook`) rewrites the worker's prompt through the harness's `updatedInput` channel, prepending a bounded `<aelfrice-worker-context>` block: L0 locked beliefs (with #1016 provenance-aware framing and reference-lock manifest bounding) plus beliefs relevant to the worker's task at the reduced auxiliary budget the Grep|Glob lane uses. Both harness channels were probed live before implementation (see the issue): `updatedInput` applies without a `permissionDecision` — so the hook never touches the user's permission flow — while `SubagentStart` was rejected for this lane (its payload carries no prompt text, so it cannot do query-aware retrieval). Fail-open everywhere (no store / malformed payload / retrieval error → byte-identical passthrough), idempotent on already-tagged prompts (nested dispatch never double-injects), and deterministic per #605. Installed by default via `aelf setup` (skip with `--no-agent-context`); runtime kill switch `AELFRICE_AGENT_CONTEXT=0`. Claude-host only for now — the Codex desired-set is unchanged pending #1056 live validation.

- **Temporal spine: ingest-time chronological edges + a dedicated retrieval lane, default-off ([#1064](https://github.com/robotrocketscience/aelfrice/issues/1064)).** The largest retrieval-coverage gain measured on this codebase, deterministic and embeddings-free: per-session `TEMPORAL_NEXT` chains written at ingest (`[ingest] write_temporal_spine` / `AELFRICE_TEMPORAL_SPINE_WRITE`), an idempotent `aelf spine backfill` for existing stores plus an `aelf doctor` spine row, and a retrieval lane (`[retrieval] use_temporal_spine` / `AELFRICE_TEMPORAL_SPINE`, node budget via `temporal_spine_budget`) that traverses the chains from the top-5 L1 seeds and appends chronological neighbours after L1. Reaches gold that shares zero salient terms with the question through chronological adjacency: confirmed **+14.6pp** gold-set coverage on LoCoMo (0.460 → 0.606; temporal +17.2pp, multi-hop +10.4pp, 10× the shuffled control), out-of-sample gain exceeding dev (+12.7pp on LongMemEval). The ingest **writer** ships default-ON (every pre-registered gate passed — G1 +14.6pp coverage, G2 trim survival + top-rank invariance, G3 latency, G5 determinism — and the G4 auto-once backfill [#1090](https://github.com/robotrocketscience/aelfrice/issues/1090) is on `main`), so fresh ingests chain the spine by default and existing stores backfill on first `aelf setup`; opt out via `AELFRICE_TEMPORAL_SPINE_WRITE=0`. The **retrieval lane** is also default-ON as of the #1107 Phase-2 cutover (see the entry above); until that landed it was wired into `retrieve_v2` only and inert on the production `retrieve()` hook path. `LaneTelemetry` gains `temporal_spine` + `temporal_spine_candidates`; `benchmarks/temporal_spine_ablation.py` is the lane's permanent coverage ablation (baseline / +spine / seeded shuffled-control).

- **Temporal-spine flip migration resolved: auto-once backfill + reversible `aelf spine clear` ([#1064](https://github.com/robotrocketscience/aelfrice/issues/1064), G4).** So the default-ON flip is effective on existing stores from day one rather than only after enough new turns accrue, the backfill now runs itself exactly once: `maybe_backfill_temporal_spine` is sentinel-gated on `~/.aelfrice/spine-backfilled` and wired into `aelf setup` beside the uv-migration. It is **gated on the writer flag**, so it stays inert while the spine is default-off (pre-flip) and re-arms — the first `aelf setup` after the flip turns the writer on fires it once, then the sentinel makes it a no-op; a fresh install is untouched until then. New **`aelf spine clear`** deletes every `TEMPORAL_NEXT` edge (beliefs untouched) as the reversibility path, and because the backfill is deterministic a later re-backfill rebuilds the identical spine. The flip itself (writer + lane defaults ON) remains a separate operator release call.

- **Codex host target for `setup` / `doctor` / `unsetup` ([#1052](https://github.com/robotrocketscience/aelfrice/issues/1052)).** `aelf setup --host codex` writes the portable aelfrice hook subset into `~/.codex/hooks.json` — merge-aware, idempotent, and refusing (without `--force`) to clobber an unparseable existing file — and prints the trust-approval steps (`/hooks` in a Codex session; `codex features enable codex_hooks`, since the upstream hooks surface is feature-flagged off by default). `aelf doctor --host codex` validates hooks.json shape, aelfrice event coverage, on-disk commands, the feature flag, and `[hooks.state]` trust coverage; `aelf unsetup --host codex` removes only aelfrice-owned entries. Trust hashes are never computed client-side — the upstream trust schema is explicitly slated to change. Compaction rebuild rides `SessionStart(source=="compact")`, the only injection channel Codex honors ([#1054](https://github.com/robotrocketscience/aelfrice/issues/1054) investigation). Bash-matcher tool hooks (memory-first shell search, pre-issue duplicate guard, commit ingest) ride along unchanged ([#1055](https://github.com/robotrocketscience/aelfrice/issues/1055)): Codex canonicalizes shell commands to `tool_name == "Bash"`, so the matchers are host-portable; only the `Grep|Glob` hook (tools that don't exist on Codex) and the host-specific memory mirror are excluded.

- **Persistent host-level auto-install opt-out ([#1053](https://github.com/robotrocketscience/aelfrice/issues/1053)).** A Codex-primary machine no longer needs `AELFRICE_NO_AUTO_INSTALL=1` inlined on every hook command: a new `opt_out_hosts` key in `~/.aelfrice/opt-out-hooks.json` (written automatically by `aelf setup --host codex` when the Claude host has no aelfrice hooks) gates the CLI-entry settings merge. Dual-host machines are left untouched; an explicit `aelf setup` clears the opt-out.

- **Valence propagation is live: feedback now flows through the belief graph ([#1058](https://github.com/robotrocketscience/aelfrice/issues/1058)).** `MemoryStore.propagate_valence` (broker-confidence-attenuated BFS, shipped in v0.1.0) had never been invoked by any production path — feedback updated exactly one belief. `apply_feedback` now walks outbound edges after every direct event and applies each attenuated delta back through itself, so downstream beliefs get a posterior update **and** a provenance-tagged `feedback_history` row (`source: propagation:<direct-source>`); propagated applications never re-propagate. Measured on a real 26k-belief store: mean fan-out 1.7 recipients per explicit event, walk latency p99 < 1 ms; implicit per-retrieval hook feedback (valence 0.1) attenuates below the 0.05 delivery threshold and propagates nothing, so hot-path volume is unchanged. Kill switch: `AELFRICE_VALENCE_PROPAGATION=0`. `aelf feedback` prints fan-out when propagation reached recipients. Also fixes a cycle bug (a loop back to the source belief returned the source in the delta map, violating the documented contract) and corrects `EDGE_VALENCE` rationale comments that cited retrieval-tier `BFS_EDGE_WEIGHTS` numbers.

- **Wide-retrieval knobs: `l1_limit` + retrieval `token_budget` are now env/TOML-configurable ([#1045](https://github.com/robotrocketscience/aelfrice/issues/1045)).** The BM25 candidate cap (`l1_limit`, default 50) is the multi-hop **recall** lever: raising it — *together with* a token budget large enough to hold the extra candidates — recovers multi-session / temporal answers that a 50-candidate slice misses (LongMemEval-S **58.8% → 68.6%** at `l1_limit=200` / `token_budget=8000`, beating the GPT-4o + LongMemEval_S paper baseline of 60.6%). Budget alone is inert: candidates cap at `l1_limit` **before** the pack trim. New `resolve_l1_limit()` / `resolve_token_budget()` follow the established precedence (`AELFRICE_L1_LIMIT` / `AELFRICE_RETRIEVAL_TOKEN_BUDGET` env → explicit kwarg → `[retrieval] l1_limit` / `token_budget` in `.aelfrice.toml` → default), wired into `retrieve()`, `retrieve_with_tiers()`, and `retrieve_v2()`. **Byte-identical to prior behaviour when unset** — the latency-sensitive per-prompt hook keeps the 50 / 2400 defaults; the knob is for retrieval-heavy / large-context callers who opt in. Fail-soft on non-numeric / non-positive values.

### Changed

- **Entity-persistence demotion flipped default-on in `retrieve_v2` ([#1096](https://github.com/robotrocketscience/aelfrice/issues/1096)).** The lane added default-off earlier this cycle now defaults on, once the G2 mixed-corpus eval ([#1103](https://github.com/robotrocketscience/aelfrice/issues/1103)) cleared the no-regression gate: durable recall held (20→20), ephemeral coordination demoted (20→3 at the tight pack budget), MRR 0.883→1.000, and the lane is recall-safe / inert on LoCoMo (whose entities are all `noun_phrase`). The resolver now also honours a `[retrieval] use_entity_persist_demote` TOML rung (precedence env → kwarg → TOML → default), so a project can opt out without an env var — matching the `use_heat_kernel` idiom and making the `CONFIG.md` precedence accurate. **Scope when flipped:** this change governed `retrieve_v2` consumers (benchmarks, the eval suite, direct callers) only — the live hook path still called the legacy `retrieve()`. The production graduation lands in this same release via the [#1107](https://github.com/robotrocketscience/aelfrice/issues/1107) Phase-3 cutover (see *Added*), so v4.0.0 ships the demotion live on every host, not just the eval surface.

- **Conversational capture is now a first-class `transcript` ingest source ([#1089](https://github.com/robotrocketscience/aelfrice/issues/1089), axis-2 prerequisite).** Passive/transcript capture previously recorded `source_kind=filesystem` in `ingest_log`, indistinguishable from project-file scanning; it now records a new `INGEST_SOURCE_TRANSCRIPT` kind. Behaviour is otherwise preserved — the retention default is unchanged (`RETENTION_FACT`), the `role=user → origin=user_transcript` undeflated-prior routing still fires, and replay-equality holds (a `transcript_v0_1.jsonl` replay-soak corpus covers the new kind, re-deriving on the `agent_inferred` path so it is stable under the `raw_meta=None` production replay). Side benefit: the user-transcript branch now keys on the transcript *source kind* rather than a bare `source_path=="transcript"` string match, so a project file literally named `transcript` can no longer spoof the undeflated user prior. This names the source so "rank curated claude-memory above conversational capture" (axis 2) becomes expressible; the ranking lever itself is a separate follow-up and this does **not** close #1089.

### Fixed

- **Retrieval exposure is audit-only by default — surfacing a belief no longer promotes it ([#1086](https://github.com/robotrocketscience/aelfrice/issues/1086)).** A hook retrieval was counted as positive evidence (+0.1 to α via `apply_feedback`), but retrieval is *exposure*, not endorsement — so whatever recurred floated above genuine knowledge. Measured on a real 24,883-belief store, junk (session scaffolding, fragments) accumulated ~3× the exposure of clean beliefs and outscored it (junk mean μ 0.554 > clean 0.446). `apply_feedback` gains an `update_posterior` flag: when `False`, the event is still written to `feedback_history` (so exposure frequency stays recoverable as the recurrence axis) but the Bayesian posterior and valence propagation are skipped. `record_retrieval` passes it through, gated by `AELFRICE_EXPOSURE_UPDATES_POSTERIOR` (default off = the fix; set `1` to restore pre-#1086 behaviour for benchmark A/B). Controlled check: a belief retrieved 50× no longer outranks a once-seen belief (both stay at prior μ=0.5); under the legacy flag the recurred belief inflated to μ=0.857. This is the exposure half of the #1086 percolation problem; the entity-persistence sink (#1096) is the demotion half.

- **Hedged conversational floats are dropped at ingest ([#1081](https://github.com/robotrocketscience/aelfrice/issues/1081)).** Passive capture stored statement-form musings ("Maybe we need X", "What if we tried Y") as factual beliefs — `classify_sentence` dropped only trailing-`?` questions, so hedged declaratives passed straight through and cluttered the store with half-thoughts. A deterministic `_is_speculative_float` predicate (mirroring `_is_question`) now sets `persist=False` on a *leading* hedge ("maybe …", "what if …", "should we …") or an unambiguous proposal-hedge phrase, placed last in the classifier so a typed requirement / correction / preference is never dropped merely for being hedged. High-precision by design — mid-sentence hedge words and bald hedge-free musings are kept and left to the retrieval / curation lanes. Pure and replay-stable.

- **Neutral-entity beliefs are no longer over-demoted by entity-persistence scoring ([#1098](https://github.com/robotrocketscience/aelfrice/issues/1098)).** `entity_persistence_scores` demoted a belief whose only entities are grounding-neutral (`noun_phrase` / `url`) by the maximum amount: the else branch registered it at `S1 = 0/(0+0+1) = 0.0`, identical to bare-number coordination junk and the `log(ε) ≈ −6.91` penalty. Durable conversational prose that extracts only noun phrases (user preferences, decisions) — the exact class #1081 / #1086 set out to protect — was demoted as hard as ephemeral `#NNN` chatter. Neutral rows are now skipped rather than registered, so a neutral-only belief never enters the map and the caller applies no demotion (matching the entity-free path); mixed beliefs are unaffected.

- **`aelf search --budget` help no longer hardcodes a stale default ([#1077](https://github.com/robotrocketscience/aelfrice/issues/1077)).** The help string read `default 2000` while the real `default=DEFAULT_TOKEN_BUDGET` is 2400; it now interpolates via argparse `%(default)s` so it can never drift from the real default again. Surfaced as code-side drift by the #1075 docs audit.

- **`doctor --prune-noise` now clears stranded capture scaffolding ([#1081](https://github.com/robotrocketscience/aelfrice/issues/1081)).** Passive-captured stores accumulate orphan section headers (a short capitalised phrase ending at a bare colon, `Recommendation:`) and shell-output echoes (`$ …`) as standalone belief rows. These pass the forward ingest filter by design — the sub-floor detector demotes an *inline* header to edge `anchor_text` (#785/#809) and drops only unanchored ones — so `is_transcript_noise` must not own the shape or it would pre-empt anchoring. A new `is_stranded_capture_noise` predicate, scoped to the store-cleanup path where every candidate is already an isolated row, extends the #1029 backfill GC so `aelf doctor --prune-noise` soft-deletes them (reversible via `valid_to`, dry-run by default, unchanged `agent_inferred`/non-locked safety scoping). High-precision: a real assertion never ends at a bare colon (`Recommendation: use X` survives) and the header pattern requires ≥2 leading characters so kappa formulas (`K:`) are safe. The FP-prone conversational-lead class (`ok …` / `yeah …`) is deliberately excluded — in a casual-writing store a real belief can open with such a marker.

- **Codex Stop events now capture real assistant text ([#1051](https://github.com/robotrocketscience/aelfrice/issues/1051)).** Codex's `transcript_path` points at a rollout JSONL that nests role and text under `payload`, a shape `_last_assistant_text` didn't recognize — every Stop event wrote an empty assistant stub, leaving belief ingestion and correction/relevance scoring inert on Codex (`aelf doctor` reported `ingest_gap`). Extraction is now per-record and shape-based: the Claude parser is unchanged, and a new Codex parser matches `response_item`/`message` records, concatenating `output_text` segments; non-message items (reasoning, tool calls) cannot shadow the answer.

- **Worktree `uv run aelf` no longer downgrades the user's installed hooks ([#1044](https://github.com/robotrocketscience/aelfrice/issues/1044)).** The #834 guard was inert for anyone who also had a `uv tool` install: `lifecycle._is_uv_tool_install()` short-circuited `True` on a filesystem-presence check (`~/.local/share/uv/tools/aelfrice/` existing *anywhere* on the box) before the correct `sys.prefix`-under-tools-root check ran — so a source worktree's `uv run aelf` was misclassified as the uv-tool install and silently rewrote `~/.claude/settings.json` hooks backwards (observed: v3.8.0 → v3.6.0, 8 hooks reset). Split the predicate: the auto-install gate now delegates to a new `lifecycle._running_from_uv_tool()` (the running-**process** check only), while `_is_uv_tool_install()` keeps its disk-presence semantics for `upgrade_advice()` (which legitimately asks "is a uv-tool install present?"). Defense in depth: `maybe_install_manifest` now refuses to stamp the hook surface **backwards** in the non-force path — a running older version skips the merge with a one-line notice instead of downgrading (an explicit `aelf setup` still re-stamps). Regression tests cover the presence-vs-process split and the never-downgrade guard.

- **`aelf onboard` no longer hides the no-API-key subagent path ([#1072](https://github.com/robotrocketscience/aelfrice/issues/1072)).** A bare `aelf onboard <path>` resolves the LLM classifier as intent (post-v1.5 default) but soft-falls to the regex classifier when a gate is unmet — missing SDK/API key, or declined/non-interactive consent — and did so **silently**, so an agent following the README's shell command landed on the crude regex store (~140 extra noise beliefs kept, collapsed type distribution) with no signal the better path existed. The silent-downgrade call sites now emit a one-line stderr pointer to `/aelf:onboard` (the no-key, no-billing subagent flow); the explicit opt-out (`--llm-classify=false`, which is also how the slash command's own `--no-subagents` fallback runs) stays silent so its verbatim output is untouched. The README and the `--llm-classify` help text now present `/aelf:onboard` as the primary onboarding path and bare `aelf onboard` as the deterministic fallback.
